# Recipe Blog

A full-stack web application for recipe creators and audiences: creators upload recipes and manage followers, audiences browse, favourite and interact. Built with a MERN-stack frontend and Node.js backend.

---

## 🧐 Overview  
This platform features two dashboards:  
- **Creator Dashboard** – Upload recipes, view followers, manage content.  
- **Audience Dashboard** – Browse recipes, mark favourites, view creator profiles.  
It demonstrates end-to-end functionality, authentication, role-based access, CRUD operations and user-centric workflows.

---

## 🚀 Built With  
- **Frontend**: React (via Vite) + CSS/SCSS  
- **Backend**: Node.js + Express  
- **Database**: MongoDB (via Mongoose)  
- **Auth & Security**: JSON Web Tokens (JWT), password hashing, environment variables  
- **API Design**: RESTful endpoints for Users, Recipes, Favourites, Followers  
- **Folder Structure**: clear separation of concerns (controllers, models, routes, middleware)

---

## ✅ Features  
- User registration & login (creator vs audience roles)  
- Role-based UI & API access  
- Creator can: upload new recipe, edit or delete existing recipe, view followers  
- Audience can: browse all recipes, favourite/un-favourite recipes, view creator profile  
- Responsive layout suitable for desktop & mobile  
- Secure backend: token-based auth, environment configs, error handlers  
- Clean codebase: modular architecture, consistent naming and structure
